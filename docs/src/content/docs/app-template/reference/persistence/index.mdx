Configure persistent storage and mount options. Each key is an identifier for a persistence item.
Supported types: persistentVolumeClaim (default), configMap, secret, nfs, emptyDir, hostPath, image, custom.
By default each item mounts to /&lt;identifier&gt;. Use globalMounts or advancedMounts to customize mount paths.

## Type

`object`

## Usage

Keys are identifiers for `persistence` instances.

```yaml
persistence:
  main:
    enabled: true
    # ... configuration

  secondary:
    enabled: true
    # ... configuration
```

## Examples

```yaml
persistence:
  config:

      - path: /config
  tmp:
    type: emptyDir
```

### Property Details

#### accessMode

AccessMode for the persistent volume. Make sure to select an access mode that is supported by your storage provider!
See https://kubernetes.io/docs/concepts/storage/persistent-volumes/#access-modes

#### advancedMounts

Explicitly configure mounts for specific controllers and containers.
Structure: advancedMounts.&lt;controller_name&gt;.&lt;container_name&gt;[].path
Use this instead of globalMounts when different controllers need different mount configurations.

#### forceRename

Override the default resource name. Mutually exclusive with prefix and suffix.

#### globalMounts

Configure mounts to all controllers and containers.
By default the persistence item will be mounted to /&lt;name_of_the_persistence_item&gt;.
Each entry supports: path, readOnly, subPath, subPathExpr, and mountPropagation.

#### prefix

Prefix to prepend to the resource name. Mutually exclusive with forceRename.

#### size

The amount of storage that is requested for the persistent volume.

#### storageClass

Storage Class for the config volume.
If set to &#39;-&#39;, dynamic provisioning is disabled.
If set to something else, the given storageClass is used.
If undefined (the default) or set to null, no storageClassName spec is set, choosing the default provisioner.

#### suffix

Suffix to append to the resource name. Defaults to the resource identifier if there are multiple items, otherwise empty. Mutually exclusive with forceRename.
