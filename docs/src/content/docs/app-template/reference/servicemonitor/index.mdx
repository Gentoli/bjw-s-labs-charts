serviceMonitor objects to be generated by the chart. Additional ServiceMonitors can be added by adding a dictionary key similar to the &#39;main&#39; ServiceMonitor.

## Type

`object`

## Usage

Keys are identifiers for `serviceMonitor` instances.

```yaml
serviceMonitor:
  main:
    enabled: true
    # ... configuration

  secondary:
    enabled: true
    # ... configuration
```

### Property Details

#### forceRename

Override the default resource name. Mutually exclusive with prefix and suffix.

#### jobLabel

The label to use to retrieve the job name from the target service&#39;s metadata. Defaults to &#39;app.kubernetes.io/name&#39;. Helm templates can be used.

#### prefix

Prefix to prepend to the resource name. Mutually exclusive with forceRename.

#### service

Service to monitor. Either &#39;serviceName&#39; or &#39;service&#39; must be specified.

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `name` | `string` | **Yes** | - |  |

#### serviceName

Reference to a Service name to monitor. Helm templates are supported. Deprecated in favor of &#39;service&#39;.

#### suffix

Suffix to append to the resource name. Defaults to the resource identifier if there are multiple items, otherwise empty. Mutually exclusive with forceRename.

#### targetLabels

Transfers labels from the Kubernetes Service onto the created metrics. See https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/api-reference/api.md#servicemonitorspec for details.
